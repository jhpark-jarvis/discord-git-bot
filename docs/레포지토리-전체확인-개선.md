## 📚 저장소 전체 확인 기능 개선 사항

**작성일**: 2026-04-27  
**개선 내용**: GitHub 계정 내 모든 저장소 조회 및 검색 기능 추가

---

## 🎯 개선 내용 요약

기존에는 `.env` 파일에 설정된 **한 개의 저장소**만 조회할 수 있었습니다.

이제 다음 기능이 추가되었습니다:

1. ✅ **내 모든 저장소 조회** (`!repo list`)
2. ✅ **저장소 검색** (`!repo search <키워드>`)

---

### 1. `src/utils/github_api.py` - GitHub API 클라이언트 확장

**추가된 메서드:**

#### `get_user_repos(username, per_page=100)`
```python
# 사용자의 모든 저장소를 GitHub API에서 조회
async def get_user_repos(self, username: str, per_page: int = 100) -> List[Dict[str, Any]]:
    """사용자의 모든 저장소 조회"""
```

**사용 예시:**
```python
api = GithubAPI(token)

# 반환값: [저장소1 정보, 저장소2 정보, ...]
```

#### `search_repos(username, keyword)`
```python
# GitHub API의 검색 기능을 사용하여 저장소 검색
async def search_repos(self, username: str, keyword: str) -> List[Dict[str, Any]]:
    """사용자의 저장소에서 검색"""
```

**사용 예시:**
```python
api = GithubAPI(token)
results = await api.search_repos("jhpark-jarvis", "discord")
# 반환값: discord 관련 저장소 목록
```

## 🏗️ 기술 세부사항

### API 호출 플로우

```
Discord 커맨드
    ↓
repo_commands.py (RepoCommands Cog)
    ↓
github_service.py (GithubService)
    ↓
github_api.py (GithubAPI - aiohttp)
    ↓
GitHub REST API
    ↓
JSON 응답
    ↓
응답 파싱 & 포매팅
    ↓
Discord Embed 생성
    ↓
Discord 채널에 메시지 전송
```

### 성능 최적화

- **최대 조회 수 제한**: 메모리/API 요청 최적화
  - `!repo list`: 최대 20개
  - `!repo search`: 최대 15개
  
- **정렬 순서**: `updated` (최근 업데이트된 순)

- **에러 처리**: API 실패 시 사용자 친화적 메시지 제공

## 🚀 향후 개선 계획

1. **페이지 네이션**
   - 저장소가 많을 경우 페이지로 나눔
   - `!repo list 2` → 2번째 페이지 표시

2. **필터링 기능**
   - 언어별 필터: `!repo list --language python`
   - 스타 수 기준: `!repo list --stars 10+`

3. **정렬 옵션**
   - `!repo list --sort stars` → 스타 많은 순
   - `!repo list --sort oldest` → 오래된 순

4. **저장소 즐겨찾기**
   - `!repo favorite <repo-name>`
   - 자주 사용하는 저장소 빠르게 접근

5. **협업 통계**
   - `!repo stats <repo-name>` → 기여자, 커밋 수 등

## 📝 코드 예시

### Python 코드에서 직접 사용

```python
from src.services.github_service import GithubService

service = GithubService("ghp_your_token_here")

# 모든 저장소 조회
repos = await service.get_user_repositories("jhpark-jarvis")
for repo in repos['repositories']:
    print(f"{repo['name']}: {repo['stars']} ⭐")

# 저장소 검색
results = await service.search_user_repositories("jhpark-jarvis", "discord")
for repo in results['results']:
    print(f"Found: {repo['full_name']}")
```

## 🔍 트러블슈팅

### Q: 저장소가 표시되지 않습니다
**A:** 
- GITHUB_TOKEN이 올바른지 확인
- 공개 저장소인지 확인
- GitHub API 레이트 리밋 확인

### Q: 검색 결과가 없습니다
**A:**
- 키워드가 2자 이상인지 확인
- 저장소명 또는 설명에 키워드 포함 여부 확인
- 다른 키워드로 시도

### Q: 응답이 느립니다
**A:**
- 저장소가 많으면 API 응답 시간 증가
- GitHub API 서버 상태 확인
- 네트워크 연결 확인
